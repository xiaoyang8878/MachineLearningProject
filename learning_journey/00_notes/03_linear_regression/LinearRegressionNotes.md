## 1. 回归问题概述

### 1.1 回归与分类的根本区别

- **回归**：预测连续数值，如房价、身高、体重。
- **分类**：预测离散标签，如鸢尾花类别、垃圾邮件、是否患病。


## 1. 任务定义：回归与分类的根本区别

| 维度 | 第二课（线性回归） | 第一课（逻辑回归） |
| :--- | :--- | :--- |
| **任务类型** | 回归（预测连续数值） | 分类（预测离散标签） |
| **输出示例** | 房价 255.5 万元 | 鸢尾花类别 0、1、2 |
| **核心公式** | `y = w₁x₁ + w₂x₂ + b` | `z = w₁x₁ + w₂x₂ + b` → Softmax → 概率 |
| **损失函数** | 均方误差（MSE）——看“距离” | 交叉熵——看“自信犯错” |


## 2. 核心公式与参数

- **单特征**：`y = kx + b`（即初中学的直线方程）。
- **多特征**：`y = w₁x₁ + w₂x₂ + b`（高维空间的"超平面"）。
- **查看参数**：`model.coef_`（权重，对应 `k` 或 `w`），`model.intercept_`（截距，对应 `b`）。

> ⚠️ **初学常见困惑：为什么 X 要 reshape？**
> 实验 B 代码中写了 `X.reshape(-1, 1)`，这是因为 sklearn 要求特征矩阵必须是**二维数组**（行=样本数，列=特征数）。
> - 1 个特征时：`-1` 自动算行数 → `6 行 1 列`   ✅
> - 不 reshape 的话：`X.shape` 是 `(6,)`（一维数组），sklearn 会报错。
> - 2 个及以上特征时：`np.column_stack((面积, 卧室数))` 直接拼出二维矩阵，无需 reshape → `60 行 2 列`。
> - **记住**：`reshape(-1, 1)` 专门用于"单特征"情况，"多特征"用 `column_stack`。

> ⚠️ **初学常见困惑：`random_state=42` 是什么？**
> 用来固定随机切分的结果。写 42 不会让模型更好，但保证**每次运行切分出的训练/测试集都一样**，方便复现对比。
> - 你可以改成任何整数（0, 1, 99…），效果一样——只要固定住就行。
> - **不写**的话，每次运行得到的数据集不同，你就无法判断效果变好是模型改进了，还是"运气好分到了简单样本"。

> 💡 **关键区分：线性回归 vs 逻辑回归优化算法**
> 线性回归使用**正规方程（Normal Equation）**一步算出最优参数——直接解最小二乘法的闭式解，**不需要迭代**，因此 `model.fit()` 瞬间完成。
> 逻辑回归使用**梯度下降（Gradient Descent）**一步步逼近最优值，需要设定学习率和迭代次数。
> 这意味着：线性回归没有"学习率"、"迭代次数"这些超参数，对初学更友好。
>
> | 对比 | 线性回归 | 逻辑回归 |
> | :--- | :--- | :--- |
> | **优化算法** | 正规方程（闭式解，一步到位） | 梯度下降（迭代逼近） |
> | **超参数** | 无（不需要学习率） | 需要学习率、迭代次数 |
> | **大数据的表现** | 百万级特征时计算 (X^T X)⁻¹ 极慢 | 大数据集仍可迭代收敛 |
> | **只有一条路** | 直接走到最优 | 可能走进局部最优（虽然逻辑回归的损失函数是凸函数） |

**课堂案例**：在实验 B 中，你给数据加了噪声后，模型学出的参数为 `k=2.87，b=12.5`。这与真实规律 `k=3，b=0` 存在偏差，原因是噪声导致模型做了一次“数学妥协”——为了照顾所有点，直线被轻微旋转并平移。

**多特征案例**：在实验 A1 中，模型同时使用“面积”和“卧室数”预测房价，学出的权重分别为 `2.5` 和 `8`，截距约为 `20`，与预设的真实规律一致。


## 3. 损失函数与评估指标

| 指标 | 含义 | 好坏判断 |
| :--- | :--- | :--- |
| **MSE（均方误差）** | 预测值与真实值距离的平方的平均值 | 越小越好。0 表示完美拟合。 |
| **R²（决定系数）** | 模型解释了目标变量多大比例的波动 | 越接近 1 越好。0.8 以上通常视为可用。 |

**课堂案例**：在实验 B 中，带噪声数据的 MSE 为正数（如 5.2），R² 约为 0.98。开根号后约等于 2.28 万元，意思是“对于测试集里的房子，模型平均猜错 2.28 万元”。


## 4. 实验B：噪声、残差与模型诊断

### 4.1 事实区分
- **真实规律（未知）**：数据产生的客观源头（如 `房价 = 3×面积`）。
- **噪声**：真实规律之上自带的随机波动（如装修溢价、测量误差）。
- **残差**：数据点与**当前模型预测值**之间的差值（`残差 = 真实值 - 预测值`）。
- **核心事实**：在不知道真实规律的情况下，你无法直接计算“噪声”，只能计算“残差”。残差是“噪声 + 模型结构性错误”的混合体。

### 4.2 通俗案例：射箭比喻
- **靶心**：真实规律（如 300 万）。
- **你瞄准的姿势**：模型公式（如 `y = kx + b`）。
- **手腕抖动**：数据自带的随机波动，称为**噪声**（发生在射箭之前）。
- **箭落点（305 万）与靶心（300 万）的偏差**：称为**残差**（发生在射箭之后）。

**核心结论**：噪声是“你手腕天生会抖”这个事实；残差是“箭落点偏离靶心”这个结果。机器永远看不到手腕抖动，它只看到落点。

### 4.3 机器没有“上帝视角”（课堂关键疑问）
- 你曾在课堂上提出一个关键观点：**“机器并不知道哪些点是噪声”**。
- 站在机器的视角：它只看到一堆散落的数据点（X, y），并不知道哪些点来自真实规律（3x），哪些点是因为装修溢价或测量误差才偏了。它唯一能做的，就是画出一条直线，让所有点到这条线的距离平方和最小。
- 因此，当你面对一份真实数据（不知道真实规律）时，你无法直接算出“纯噪声”。你只能算出“残差”（数据离你这条线的距离）。

### 4.4 如何通过残差判断模型好坏
- **残差随机分布在零线上下（无走势）** → 模型已基本捕获规律，残差近似等于噪声。无需进一步改进。
- **残差呈现明显形状（如U形、波浪形）** → 残差中仍残留未捕获的规律，模型欠拟合，需更换或升级模型。

**课堂案例（残差图对比）**：
- 场景 1（线性数据）：残差点在零线上下随机散落，呈“雾状”，没有走势 → 模型合格。
- 场景 2（曲线数据）：残差点呈现明显的 U 形 → 直线模型无法捕获曲线规律，模型不合格。

### 4.5 如果我不知道真实规律，该怎么判断？（实操方法）
在现实工作中，你永远不会知道真实规律是什么。因此，你无法直接回答“这个偏差里有多少是噪声，有多少是模型没学好”。

**实操做法**：
1. 训练一个基准模型（如线性回归）。
2. 画出残差图，观察是否有形状。
3. 如果有形状（如U形），说明残差里还藏着信号，用非线性模型（如决策树）继续试。
4. 如果换了几种模型后，残差图都近似随机分布，就认为模型已尽力，残差≈噪声。

**结论**：残差图是你的“X光机”——它不告诉你噪声有多大，但它能告诉你**模型中还有没有没榨干的信号**。


## 5. 实验A1：多特征扩展（平面拟合）

- 将 `X` 从一列（面积）扩展为两列（面积 + 卧室数）。
- 模型从“寻找最佳直线”升级为“寻找最佳平面”（三维）。
- 代码结构不变：`model.fit(X, y)` 自动适配特征数量。
- 特征继续增加至3个以上时，模型寻找“最佳超平面”（人类不可视，但数学原理一致）。


## 6. 实验A2：线性回归与逻辑回归在梯度下降中的本质区别

| 对比维度 | 线性回归（MSE） | 逻辑回归（交叉熵） |
| :--- | :--- | :--- |
| **误差输入范围** | 任意实数（-∞, +∞），如 `300万 - 280万 = 20万` | 固定范围 [-1, 1]，因预测概率只能在 0~1 之间 |
| **对极端值的敏感度** | 非常敏感。离群点（如天价房）会强行拖拽直线 | 不敏感。极端样本的梯度会趋近于0，模型自动忽略它 |
| **梯度关注区域** | 盯着“物理距离”远的点 | 盯着“决策边界（50%概率）”附近的模糊点 |

**课堂案例（异常值拖拽直线）**：
- 在线性回归中，如果你把一栋 50 平的房子房价从 150 万改成 3000 万（异常值），拟合出的 `k` 会大幅下降，`b` 会大幅上升，直线被强行拽向异常点，导致其他正常房子的预测都偏了。
- 在逻辑回归中，如果你把一个标签为 0 的样本的特征改得极其极端（如花瓣长度设为 100 厘米），模型预测概率会极度接近 0 或 1，此时梯度趋近于零，模型会“放弃”这个极端点，继续把注意力放在决策边界（概率在 50% 左右）的样本上。


## 7. 代码流水线（标准格式）

```python
# 1. 准备数据
X = np.array([...]).reshape(-1, 1)   # 或 X = np.column_stack((特征1, 特征2))
y = np.array([...])

# 2. 切分数据（闭卷考试）
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# 3. 创建模型并训练
model = LinearRegression()
model.fit(X_train, y_train)

# 4. 查看参数
print(model.coef_, model.intercept_)

# 5. 预测与评估
y_pred = model.predict(X_test)
mse = mean_squared_error(y_test, y_pred)
r2 = r2_score(y_test, y_pred)

# 6. （诊断）画残差图
residual = y_test - y_pred
plt.scatter(y_pred, residual)
plt.axhline(0, color='red', linestyle='--')
```


## 8. 本课核心要点

1. 回归任务是预测**连续数字**，损失函数看“**距离**”（MSE）。
2. **噪声**是数据自带的先天波动；**残差**是模型拟合后的后天偏差。你只能计算残差，并通过残差图的形状判断模型是否合格。
3. **机器没有上帝视角**：它不知道哪些点是噪声，它只对输入的数据执行最小化距离平方和的计算。
4. 线性回归对异常值（离群点）**高度敏感**，这是它的天然弱点。
5. 多特征扩展只需改变 `X` 的列数，代码其余部分不变。
6. 线性回归的梯度盯着“物理距离”，误差越大冲得越猛；逻辑回归的梯度盯着“边界模糊度”，只关心概率在 50% 附近的样本，极端点反而被忽略。


## 9. 与第三课的衔接

线性模型（包括逻辑回归和线性回归）的边界或平面是**直线**或**平面**。当数据规律是曲线或环形时，线性模型会失效。第三课将引入**决策树**，它不依赖梯度下降，通过“连环问答”的方式处理非线性问题。
