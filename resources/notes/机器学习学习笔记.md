## 0. 数学基础

### 0.1 线性代数
- **标量、向量、矩阵、张量**：标量是单个数值；向量是一维数组；矩阵是二维数组；张量是更高维的数组。
- **向量点积**：$\mathbf{a} \cdot \mathbf{b} = \sum_i a_i b_i$，用于计算相似度。
- **矩阵乘法**：$\mathbf{C} = \mathbf{A} \mathbf{B}$，要求 $\mathbf{A}$ 的列数等于 $\mathbf{B}$ 的行数。结果矩阵 $C_{ij} = \sum_k A_{ik} B_{kj}$。
- **转置**：$(\mathbf{A}^\top)_{ij} = \mathbf{A}_{ji}$。
- **逆矩阵**：$\mathbf{A}^{-1} \mathbf{A} = \mathbf{I}$，用于求解线性方程组。
- **特征值与特征向量**：$\mathbf{A} \mathbf{v} = \lambda \mathbf{v}$，用于 PCA。
- **奇异值分解（SVD）**：$\mathbf{A} = \mathbf{U} \mathbf{\Sigma} \mathbf{V}^\top$，任何矩阵可分解。

### 0.2 微积分
- **导数**：$f'(x) = \lim_{h \to 0} \frac{f(x+h)-f(x)}{h}$。
- **偏导数**：$\frac{\partial f}{\partial x_i}$，固定其他变量。
- **梯度**：$\nabla f = \left( \frac{\partial f}{\partial x_1}, \dots, \frac{\partial f}{\partial x_n} \right)$，指向函数上升最快的方向。
- **链式法则**：$\frac{d}{dx} f(g(x)) = f'(g(x)) g'(x)$，反向传播基础。

### 0.3 概率与统计
- **随机变量**：离散（掷骰子）或连续（身高）。
- **常见分布**：伯努利、二项、高斯（正态）、指数。
- **期望** $E[X] = \sum x P(x)$ 或 $\int x f(x) dx$；**方差** $Var(X)=E[(X-E[X])^2]$。
- **贝叶斯定理**：$P(A|B) = \frac{P(B|A)P(A)}{P(B)}$。
- **最大似然估计（MLE）**：$\theta_{MLE} = \arg\max_\theta P(D|\theta)$。

### 0.4 优化基础
- **凸函数**：局部最小值即全局最小值。
- **梯度下降**：$\theta \leftarrow \theta - \eta \nabla_\theta J(\theta)$，其中 $\eta$ 是学习率。  
  - 批量梯度下降：使用全部样本计算梯度。  
  - 随机梯度下降（SGD）：使用单个样本。  
  - 小批量梯度下降：使用一小批样本（最常用）。
- **拉格朗日乘子**：用于约束优化，如 SVM 对偶问题。

## 1. 机器学习概述

### 1.1 定义
- **机器学习**：从数据中学习算法，构建模型进行预测。
- 本质：模拟人决策过程。

### 1.2 基本概念
- **输入 $x \in X$**：特征属性。  
- **输出 $y \in Y$**：目标值。  
- **目标函数 $f: X \to Y$**：理想映射（未知）。  
- **训练数据 $D = \{(x_1,y_1),...,(x_n,y_n)\}$**。  
- **假设函数 $g: X \to Y$**：学到的模型。

### 1.3 拟合问题
- **拟合**：模型符合训练数据。  

- **鲁棒性**：模型对异常值、噪声或数据微小变化不敏感，输出稳定。

- **泛化能力**：模型在未见过的测试数据上表现良好（训练误差和测试误差都低）。
  *注意：鲁棒性强有助于泛化，但两者不等价。*

- **过拟合**：过度记忆训练数据，泛化差。**训练效果好，测试效果差 -- 伪学霸。  -- 训练数据存在标签的冗余或者太多无效标签**

  常见原因：

  - **训练数据太少**

    样本量不足以覆盖真是分布的多样性，模型容易记住噪声。

  - **模型过于复杂**

    比如参数数量远超样本量（如用高阶多项式拟合几个点，或用很深的神经网络训练小数据集）。

  - **特征过多**

    加入了大量无关或冗余特征，模型学到的是特征与标签之间的偶然关系。

  - **训练时间过长（深度学习）**

    反复迭代多次后，模型开始拟合训练集中的噪声而非信号。

  - **数据泄露**

    训练数据中意外包含了验证集/测试集的信息（如归一化时用了全局统计量），导致验证误差看似很低，但泛化很差。

- **欠拟合**：未捕捉到数据规律。**训练效果不好 -- 学渣**

  常见原因：

  - **模型过于简单**

    例如用线性回归拟合明显非线性的数据，或决策树深度太浅。

  - **特征不足或质量差**

    没有提供有预测能力的特征（如用月份预测股票涨跌，但缺少关键因子）。

    特征与目标之间本无关联（如用随机噪声作为特征），模型无法学习到规律。

  - **正则化过强**

    L1/L2惩罚系数过大，导致模型几乎被“压扁”到0，无法拟合基本关系。

  - **训练不充分**

    迭代次数太少

    学习率过低导致收敛缓慢

    提前停止过早

  - **数据预处理错误**

    例如误删了重要特征、错误归一化导致信号失真。

    

### 1.4 与 AI、深度学习的关系
- 深度学习（多层神经网络） ⊆ 机器学习 ⊆ 人工智能。

### 1.5 与数据分析、数据挖掘的区别
- **数据分析**：统计分析，提取信息。  
- **数据挖掘**：从大数据中挖掘隐藏模式。  
- **机器学习**：算法自动学习。

### 本章思考题
1. 过拟合和欠拟合分别如何从训练/验证误差上判断？

   - **过拟合在训练的误差较小，但是在验证的误差明显高于训练误差（且泛化性能差）。说明模型背下了训练数据，但是并没有学到普遍规律。**

   - **欠拟合无论是训练误差还是验证误差都很高。说明模型连训练数据都没学好，能力不足。**

     **理想情况是训练误差和验证误差都较低，且两者接近。如果出现训练误差很低、验证误差也很低但略高，这是正常情况，不算过拟合。**

2. 举一个既可用分类又可用回归解决的商业问题，并说明两者的不同输出。

   - **在一个预测股票市场趋势的模型中，如果需要预测股票的涨跌情况，那么可以使用分类**
   - **如果需要预测股票的股价是多少，那么可以使用回归**

## 2. 数据预处理与特征工程

### 2.1 数据收集与存储
- 来源：业务数据库、日志、第三方 API。  
- 存储：MySQL、HDFS、Kafka、Redis 等。  
- 公开数据集：UCI、Kaggle、天池。

### 2.2 数据清洗与转换

#### 缺失值处理
- **删除**：缺失率 > 50% 的行或列。  
- **填充**：均值/中位数/众数；前向/后向填充（时间序列）；模型预测填充（KNN、回归）。  
- **标记缺失**：新增指示列，对树模型有效。

#### 异常值检测
- **IQR 法**：$[Q1-1.5\times IQR,\ Q3+1.5\times IQR]$ 之外为异常。  
- **Z-score**：$|Z| > 3$ 为异常（假设正态）。  
- **盖帽法**：替换为边界值。

#### 特征缩放
- **归一化（Min-Max）**：$x' = \frac{x - \min}{\max - \min}$，范围 $[0,1]$。  
- **标准化（Z-score）**：$x' = \frac{x - \mu}{\sigma}$，均值为 0，方差 1。  
- 需要缩放的算法：SVM、KNN、神经网络、PCA。  
- 不需要的算法：树模型、朴素贝叶斯。

**代码示例**：
```python
from sklearn.preprocessing import StandardScaler, MinMaxScaler
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)  # X is numpy array or dataframe
```

#### 特征编码
- **One-Hot 编码**：将无序类别转为二进制向量。  
- **标签编码**：将类别映射为整数（适合有序类别）。  
- **目标编码**：用类别对应的目标均值编码（需交叉验证防过拟合）。

**代码示例**：
```python
from sklearn.preprocessing import OneHotEncoder
encoder = OneHotEncoder(sparse_output=False)
X_encoded = encoder.fit_transform(X_categorical)
```

#### 特征选择
- **过滤法**：方差阈值、卡方检验、互信息。  
- **包裹法**：递归特征消除（RFE）。  
- **嵌入法**：L1 正则化（Lasso）、树模型特征重要性。

**代码示例**：
```python
from sklearn.feature_selection import SelectKBest, chi2
selector = SelectKBest(chi2, k=10)
X_new = selector.fit_transform(X, y)
```

#### 特征降维
- **PCA**：线性降维，投影到方差最大的方向。  
- **t-SNE**：非线性，用于可视化（2/3 维）。  
- **UMAP**：类似 t-SNE，更快。

**代码示例**：
```python
from sklearn.decomposition import PCA
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X)
```

#### 数据不平衡处理
- **过采样**：SMOTE（合成少数类样本）。  
- **欠采样**：随机删除多数类样本。  
- **代价敏感学习**：设置 `class_weight='balanced'`。

**代码示例**：
```python
from imblearn.over_sampling import SMOTE
smote = SMOTE()
X_resampled, y_resampled = smote.fit_resample(X, y)
```

## 3. 有监督学习

- 用已知某种或某些特性的样本作为训练集，以建立一个数学模型，再用已建立的模型来预测未知样本，此种方法被称为有监督学习，是最常用的一种机器学习方法。是从**标签化**训练数据集中推断出模型的机器学习任务。
- 总的来说，就是对于输入数据X能预测Y
- 有监督学习可以分为回归任务和分类任务

### 3.1 任务类型
- **分类**：离散标签（如良性/恶性）。  
- **回归**：连续标签（如房价）。

### 3.2 判别式 vs 生成式

- **判别式模型(Discriminative Model**)：直接对条件概率**p(y|x)**进行建模，常见判别模型有：**Logistic回归、决策树、支持向量机SVM、k近邻、神经网络**等；
- **生成式模型(Generative Model)**：对联合分布概率**p(x,y)**进行建模，常见生成式模型有：**隐马尔可夫模型HMM**、**朴素贝叶斯模型**、**高斯混合模型GMM**、**LDA（Latent Dirichlet Allocation，主题模型）**等；
- 生成式模型更普适；判别式模型更直接，目标性更强
- 生成式模型关注数据是如何产生的，寻找的是数据分布模型；判别式模型关注的数据的差异性，寻找的是分类面
- 由生成式模型可以产生判别式模型，但是由判别式模式没法形成生成式模型
- 总的来说，就是对于输入数据X能发现什么

| 类型 | 建模对象 | 常见算法 |
|------|----------|----------|
| 判别式 | $P(y\|x)$ | 逻辑回归、SVM、KNN、决策树、神经网络 |
| 生成式 | $P(x,y)$ | 朴素贝叶斯、HMM、GMM、LDA（主题模型） |

### 3.3 核心算法

#### 线性回归
- **模型**：$y = \mathbf{w}^\top \mathbf{x} + b$，也可写作 $y = \mathbf{w}^\top \mathbf{x}$ 其中 $\mathbf{x}$ 包含常数项 1。
- **损失函数**：均方误差 $MSE = \frac{1}{n}\sum_{i=1}^n (y_i - \hat{y}_i)^2$。
- **闭式解**：$\mathbf{w} = (X^\top X)^{-1} X^\top \mathbf{y}$，其中 $X$ 是 $n \times d$ 矩阵（每行一个样本，包含偏置列）。
- **梯度下降更新**：$\mathbf{w} \leftarrow \mathbf{w} - \eta \frac{2}{n} X^\top (X\mathbf{w} - \mathbf{y})$。
- **评估**：MSE、RMSE、MAE、$R^2$。

**代码示例**：
```python
from sklearn.linear_model import LinearRegression
model = LinearRegression()
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
print("R^2:", model.score(X_test, y_test))
```

#### 逻辑回归
- **二分类**：$P(y=1|x) = \sigma(\mathbf{w}^\top \mathbf{x} + b)$，$\sigma(z)=\frac{1}{1+e^{-z}}$。
- **损失函数**：对数损失（交叉熵）$-\frac{1}{n}\sum [y_i \log \hat{y}_i + (1-y_i)\log(1-\hat{y}_i)]$。
- **多分类**：Softmax 回归，输出每个类别的概率。

**代码示例**：
```python
from sklearn.linear_model import LogisticRegression
model = LogisticRegression(max_iter=1000)
model.fit(X_train, y_train)
print("Accuracy:", model.score(X_test, y_test))
```

#### 💡 线性回归 vs 逻辑回归：核心区别速查表

虽然名字里都有"回归"，但它们是**完全不同的两种算法**：

| 对比维度 | 线性回归 | 逻辑回归 |
|---------|---------|----------|
| **问题类型** | 回归（预测连续值） | 分类（预测离散类别） |
| **典型应用** | 房价预测、销量预测、温度预测 | 垃圾邮件检测、疾病诊断、点击率预测 |
| **输出范围** | (-∞, +∞) 任意实数 | [0, 1] 概率值 → 映射为类别标签 |
| **激活函数** | 无 | Sigmoid（把任意实数压缩到0~1） |
| **损失函数** | MSE（均方误差/最小二乘） | 交叉熵（对数似然损失） |
| **优化方法** | 可直接求解析解（闭式解） | 需要梯度下降迭代求解 |
| **评估指标** | MSE、RMSE、MAE、R² | 准确率、精确率、召回率、F1、AUC |

**通俗理解：**
- **线性回归**：像预测明天温度是多少度（25.3°C），输出是一个具体数值。
- **逻辑回归**：像判断明天会不会下雨（是/否），输出是概率（80%会下雨），然后以50%为阈值决定类别。

**为什么逻辑回归叫"回归"？**
- 历史原因：它的内部计算确实是回归（线性组合 `w·x + b`）
- 只是最后加了一个 **Sigmoid 函数**把结果变成概率
- 早期统计学家把它看作"对概率的对数几率做回归"，所以保留了这个名字
- **实际上它是分类算法！**

#### K近邻（KNN）

**核心思想**：物以类聚，人以群分。新样本的类别由最近的 K 个邻居投票决定。

**工作原理**：
1. 计算待预测样本与所有训练样本的距离
2. 找出距离最近的 K 个邻居
3. **分类任务**：K 个邻居中哪个类别最多，就预测为该类别（多数表决）
4. **回归任务**：K 个邻居的目标值取平均，作为预测结果

**距离度量**：最常用欧氏距离（就是两点之间的直线距离公式）
$$d(x, x') = \sqrt{\sum_{j=1}^{d} (x_j - x'_j)^2}$$

**KNN 三要素**：

| 要素 | 说明 | 影响 |
|------|------|------|
| **K值选择** | 通过交叉验证确定最佳K | K太小 → 过拟合；K太大 → 欠拟合 |
| **距离度量** | 常用欧氏距离、曼哈顿距离 | 不同距离适用不同场景 |
| **决策规则** | 分类用投票，回归用平均 | 可加权（距离越近权重越大） |

**加权投票/平均法**：
- 不是简单投票，而是给每个邻居分配权重 $w = \frac{1}{d}$（距离的倒数）
- 距离越近的邻居，影响力越大
- **例子**：3个红色点距离为2，2个黄色点距离为1
  - 红色总权重 = 3 × 0.5 = 1.5
  - 黄色总权重 = 2 × 1 = 2
  - 预测为黄色（因为黄色权重更大）

**优缺点**：
- ✅ 优点：简单直观，无需训练阶段，天然支持多分类
- ❌ 缺点：需要特征缩放，高维数据效果差（维数灾难），预测速度慢
-  加速技巧：使用 KD 树或 Ball Tree 加速最近邻搜索

**代码示例**：
```python
from sklearn.neighbors import KNeighborsClassifier
model = KNeighborsClassifier(n_neighbors=5)  # K=5
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
```

#### 决策树
- **划分标准**：  
  - 信息增益（ID3）：$Gain = H(S) - \sum \frac{|S_v|}{|S|} H(S_v)$，其中 $H$ 为熵。  
  - 信息增益率（C4.5）。  
  - 基尼系数（CART）：$Gini = 1 - \sum p_k^2$。
- **剪枝**：预剪枝（提前停止）、后剪枝（合并叶子）防过拟合。

**代码示例**：
```python
from sklearn.tree import DecisionTreeClassifier
model = DecisionTreeClassifier(max_depth=5, min_samples_split=10)
model.fit(X_train, y_train)
```

#### 支持向量机（SVM）
- **核心思想**：最大化分类间隔。
- **硬间隔**：数据线性可分时，$ \min_{\mathbf{w},b} \frac{1}{2}\|\mathbf{w}\|^2$ 约束 $y_i(\mathbf{w}^\top \mathbf{x}_i+b) \ge 1$。
- **软间隔**：引入松弛变量 $\xi_i$，目标 $ \frac{1}{2}\|\mathbf{w}\|^2 + C \sum \xi_i$。
- **核技巧**：将数据映射到高维空间，内积可用核函数 $K(x_i, x_j)$ 计算。常用核：线性、多项式、RBF（高斯）、sigmoid。
- **对偶形式**：$\max_\alpha \sum \alpha_i - \frac{1}{2}\sum_{i,j} \alpha_i \alpha_j y_i y_j K(x_i, x_j)$，约束 $0\le \alpha_i\le C$，$\sum \alpha_i y_i=0$。

**代码示例**：
```python
from sklearn.svm import SVC
model = SVC(kernel='rbf', C=1.0, gamma='scale')
model.fit(X_train, y_train)
```

#### 朴素贝叶斯
- **贝叶斯公式**：$P(y|x) \propto P(y) \prod_{i} P(x_i|y)$，假设特征条件独立。
- **高斯朴素贝叶斯**：$P(x_i|y) \sim \mathcal{N}(\mu_{y,i}, \sigma_{y,i}^2)$，适用于连续特征。
- **伯努利/多项式**：适用于离散特征（如文本词频）。

**代码示例**：
```python
from sklearn.naive_bayes import GaussianNB
model = GaussianNB()
model.fit(X_train, y_train)
```

#### 集成学习
- **Bagging**：并行训练，投票/平均。代表：**随机森林**（决策树 + 随机特征选择），降低方差。

## 4. 无监督学习

- 与监督学习相比，无监督学习的训练集中**没有人为的标注的结果**，在非监督的学习过程中，**数据并不被特别标识**，学习模型是为了**推断出数据的一些内在结构**。
- 无监督学习试图学习或者提取数据背后的数据特征，或者从数据中抽取出重要的特征信息，常见的算法有聚类、降维、文本处理(特征抽取)等。
- 无监督学习一般是作为有监督学习的前期数据处理，功能是从原始数据中抽取出必要的标签信息。
- 无监督学习本身也有独立目标，如**异常检测、数据可视化、发现隐藏结构**，不一定要服务于后续有监督任务。

### 4.1 聚类
- **K-Means**：将样本划分为 K 个簇，最小化簇内平方和。  
  - 算法：随机初始化质心，分配样本，更新质心，重复。  
  - 选择 K：肘部法则（SSE 下降拐点）、轮廓系数。  
- **DBSCAN**：基于密度，无需指定 K，可发现任意形状簇，处理噪声。参数 $\varepsilon$ 和 `min_samples`。  
- **层次聚类**：凝聚或分裂，输出树状图。  
- **高斯混合模型（GMM）**：假设数据由多个高斯分布混合生成，EM 算法估计参数，可给出软聚类。

**代码示例**：

### 4.2 降维
- **PCA**：线性降维，保留最大方差方向。步骤：中心化 → 协方差矩阵 → 特征值分解 → 取前 k 个特征向量。  
- **t-SNE**：非线性，保留局部结构，用于可视化。  
- **UMAP**：保留全局+局部，更快。

**代码示例**：
```python

```

### 4.3 关联规则
- **Apriori**：发现频繁项集，生成关联规则（置信度、提升度）。  
- **FP-Growth**：更高效，无需生成候选项集。

### 4.4 异常检测
- **孤立森林**：异常点路径短。  
- **One-Class SVM**：学习边界，边界外为异常。  
- **统计方法**：Z-score、箱线图。

## 6. 强化学习

强化学习(Reinforcement Learning, RL)，又称再励学习、评价学习或者增强学习，是机器学习的范式和方法论之一，用于描述和解决智能体(agent)在与环境的交互过程中通过学习策略以达成回报最大化或实现特定目标的问题。

### 基本概念
- **智能体（Agent）**、**环境（Environment）**、**状态（State） $s$**、**动作（Action） $a$**、**奖励（Reward） $r$**、**策略（Policy） $\pi(a|s)$**。
- **价值函数**：$V^\pi(s) = E[\sum_{t=0}^\infty \gamma^t r_t | s_0=s, \pi]$（状态价值）。  
- **Q 函数**：$Q^\pi(s,a) = E[\sum_{t=0}^\infty \gamma^t r_t | s_0=s, a_0=a, \pi]$（动作价值）。

### 核心算法

#### Q-learning（表格型）
- **更新公式**：$Q(s,a) \leftarrow Q(s,a) + \alpha \left[ r + \gamma \max_{a'} Q(s',a') - Q(s,a) \right]$。
- **Off-policy**：使用 $\epsilon$-greedy 探索。

#### 深度 Q 网络（DQN）
- 用神经网络近似 $Q(s,a)$，使用经验回放（Experience Replay）和目标网络（Target Network）稳定训练。

#### 策略梯度
- 直接优化策略 $\pi_\theta(a|s)$，梯度 $\nabla_\theta J(\theta) = E_{\pi_\theta} [ \nabla_\theta \log \pi_\theta(a|s) \cdot Q^{\pi_\theta}(s,a) ]$。
- 代表：REINFORCE、PPO（Proximal Policy Optimization）。

**代码示例（简单 Q-learning 伪代码）**：
```python
# 初始化 Q 表
Q = defaultdict(lambda: np.zeros(n_actions))
for episode in range(num_episodes):
    s = env.reset()
    while not done:
        a = epsilon_greedy(Q[s], epsilon)
        s_next, r, done = env.step(a)
        Q[s][a] += lr * (r + gamma * np.max(Q[s_next]) - Q[s][a])
        s = s_next
```

### 7.2 评估指标

#### 分类指标（预测类别标签）
- **准确率**：$\frac{TP+TN}{Total}$。  
- **精确率**：$\frac{TP}{TP+FP}$。  
- **召回率**：$\frac{TP}{TP+FN}$。  
- **F1 分数**：$2 \cdot \frac{P \cdot R}{P+R}$。  
- **ROC/AUC**：以 FPR 为横轴，TPR 为纵轴的曲线下面积。

**代码示例**：
```python

```

#####  1.搞懂“混淆矩阵”

对于二分类（正例 = 我们关注的类别，如“垃圾邮件”），预测结果有四种情况：

|              | 预测为“正例” | 预测为“负例”   |
| ------------ | ------------ | -------------- |
| 真实是“正例” | TP（命中）   | FN（漏报）     |
| 真实是“负例” | FP（误报）   | TN（正确拒绝） |

- **TP**：猜对了正例
- **TN**：猜对了负例
- **FP**：把负例错当正例（虚惊一场）
- **FN**：把正例漏掉了（危险）

##### 2. 准确率（Accuracy）

- **公式**：`(TP + TN) / (所有样本)`
- **大白话**：总共猜对了多少个（包括正和负）。
- **例子**：100封邮件，90封正常10封垃圾，模型全猜正常 → 准确率90%，但一个垃圾都没识别出来。
- **缺点**：**类别不平衡时很坑**。比如只有1%的正例，模型全猜负例也能有99%准确率，但毫无用处。

##### 3. 精确率（Precision）

- **公式**：`TP / (TP + FP)`
- **大白话**：模型说是正例的那些里面，有多少是真正的正例。
- **例子**：模型把10封邮件标为垃圾，其中8封真的是垃圾 → 精确率=80%。
- **关注点**：怕“误报”时看这个（比如把重要邮件误判为垃圾）。

##### 4. 召回率（Recall）

- **公式**：`TP / (TP + FN)`
- **大白话**：真正的正例里面，模型找出了多少。
- **例子**：总共有20封垃圾邮件，模型只找出了15封 → 召回率=75%。
- **关注点**：怕“漏报”时看这个（比如癌症筛查，漏掉一个可能致命）。

##### 5. F1分数

- **公式**：`2 × (精确率 × 召回率) / (精确率 + 召回率)`
- **大白话**：精确率和召回率的**调和平均**，兼顾两者。
- **什么时候用**：当你想同时平衡误报和漏报，且类别不平衡时。
- **例子**：精确率80%，召回率75%，F1≈77.4%。

##### 6. ROC 曲线 & AUC

- **ROC曲线**：横轴是“假正率”（FP / (FP+TN)），纵轴是“召回率”（TPR）。曲线越靠近左上角越好。
- **AUC**：曲线下的面积，取值0~1。AUC=0.5表示瞎猜，AUC=1表示完美。
- **优点**：不依赖阈值设置，衡量模型整体排序能力。
- **适用**：需要比较模型优劣，或对概率输出进行评估。

##### **分类指标速选表**

| 业务场景       | 最看重指标 | 解释                       |
| -------------- | ---------- | -------------------------- |
| 垃圾邮件过滤   | 精确率     | 不希望误把正常邮件标为垃圾 |
| 疾病筛查       | 召回率     | 宁可误报，也不能漏掉病人   |
| 两者都重要     | F1分数     | 平衡误报和漏报             |
| 类别极度不平衡 | AUC        | 评估排序能力               |
| 各类别比例均衡 | 准确率     | 简单直观                   |

#### 回归指标（预测连续数值）

- **MSE**：$\frac{1}{n}\sum (y_i - \hat{y}_i)^2$。  
- **RMSE**：$\sqrt{MSE}$。  
- **MAE**：$\frac{1}{n}\sum |y_i - \hat{y}_i|$。  
- **$R^2$**：$1 - \frac{\sum (y_i-\hat{y}_i)^2}{\sum (y_i-\bar{y})^2}$。

##### 1.MAE(平均绝对误差)**

- **大白话**：平均每个预测值对比真实值差了多少（绝对值，不看方向）。
- **例子**：真实租赁量 [1000, 2000]，预测 [1100, 1800]，误差分别是 100 和 200，平均 MAE = 150 辆。
- **优点**：容易理解，单位跟原始数据一样。
- **缺点**：不区分是预测高了还是低了。
- **怎么算好**：越小越好。如果 MAE 比目标变量的平均值小很多（比如 10% 以内），模型不错。

##### 2.**MSE(均方误差)**

- **大白话**：把每个误差**平方**后再平均。
- **例子**：同上，误差 100 和 200，平方后 10000 和 40000，平均 MSE = 25000。
- **注意**：它的单位是原始单位的平方，所以数值通常很大，不直接读。
- **优点**：放大了大误差的影响，适合惩罚那些离谱的预测。
- **缺点**：不直观，受异常值影响大。

##### 3.**RMSE(均方根误差)**

- **大白话**：把 MSE 开个平方根，单位回到原始单位。
- **例子**：MSE=25000，RMSE≈158 辆。
- **比较**：如果 RMSE 明显大于 MAE，说明存在一些误差非常大的样本。
- **推荐**：用 RMSE 替代 MSE 来直观比较。

##### 4.**MAPE(平均绝对百分比误差)**

- **大白话**：平均误差占了真实值的百分之几。
- **例子**：真实 1000，预测 1100 → 误差 10%；真实 2000，预测 1800 → 误差 10%；平均 MAPE = 10%。
- **优点**：无量纲，可跨不同规模的数据比较，业务上最直观。
- **注意**：真实值接近 0 时会出问题（除法溢出）。

##### 5.**R²（决定系数）**

- **大白话**：模型能解释目标变量变动的比例。
- **例子**：R²=0.8 表示模型可以解释 80% 的波动，剩下 20% 是模型没抓到的。
- **取值**：最大为 1，越小越差，可以为负数（比直接用平均值还烂）。
- **优点**：归一化，不受数据量纲影响。

##### 回归指标速选表

| 你的需求       | 推荐指标                  |
| -------------- | ------------------------- |
| 向老板汇报     | MAE 或 MAPE               |
| 惩罚巨大偏差   | RMSE                      |
| 对比不同数据集 | MAPE 或 R²                |
| 简单判断好坏   | MAPE（百分比小于10%算好） |

### 7.3 通用思路——如何判断模型好不好？

#### 1. 不看绝对值，看相对值

- 回归：MAPE < 10% 通常不错；10%~20% 可接受；>30% 很差。
- 分类：精确率/召回率/F1 大于 0.8 算良好，0.9 以上优秀。

#### 2. 一定要对比“傻瓜基线”

- 回归基线：用训练集**均值**预测。
- 分类基线：用训练集**多数类**预测（比如全猜负例）。  
  如果你的模型还不如基线，说明没学到东西。

#### 3. 画图检查

- 回归：画**预测值 vs 真实值**散点图（应在 y=x 线附近）。
- 分类：画**混淆矩阵**的热力图，一眼看出错在哪类。

#### 4. 根据业务选择指标

- 不要贪多，选1~2个核心指标即可。
- 回归：MAE + MAPE 足够。
- 分类：精确率+召回率 或 F1，必要时看 AUC。

## 8. 神经网络与深度学习

### 8.1 基础结构
- **感知机**：单层，线性分类。  
- **多层感知机（MLP）**：输入层 + 隐藏层 + 输出层。万能逼近定理。

### 8.2 激活函数
- **Sigmoid**：$\sigma(x)=1/(1+e^{-x})$，范围 (0,1)，易梯度消失。  
- **Tanh**：$\tanh(x)$，范围 (-1,1)，零中心化。  
- **ReLU**：$\max(0,x)$，计算快，缓解梯度消失，但可能神经元死亡。  
- **Softmax**：多分类输出，$\text{softmax}(z)_i = e^{z_i}/\sum_j e^{z_j}$。

### 8.3 损失函数
- 回归：MSE。  
- 分类：交叉熵 $-\sum y_i \log \hat{y}_i$。

### 8.4 反向传播（推导示例）
对于单隐藏层网络，输出 $a = \sigma(W_2 \cdot \sigma(W_1 x + b_1) + b_2)$，损失 $L = \frac{1}{2}(a-y)^2$。  
- 计算 $\delta_2 = (a-y) \cdot \sigma'(z_2)$，$z_2 = W_2 h + b_2$。  
- 计算 $\delta_1 = (W_2^\top \delta_2) \cdot \sigma'(z_1)$，$z_1 = W_1 x + b_1$。  
- 梯度：$\frac{\partial L}{\partial W_2} = \delta_2 h^\top$，$\frac{\partial L}{\partial W_1} = \delta_1 x^\top$。

### 8.5 优化器
- **SGD**：小批量更新。  
- **Momentum**：加入惯性 $v_t = \beta v_{t-1} + (1-\beta)\nabla L$。  
- **Adam**：自适应学习率，结合 Momentum 和 RMSprop，最常用。

### 8.6 常用架构
- **CNN**：卷积层（提取局部特征）、池化层（降维）、全连接层。适用于图像。  
- **RNN/LSTM**：处理序列，LSTM 用门控解决长期依赖。  
- **Transformer**：自注意力机制，取代 RNN 成为 NLP 主流。

**代码示例（使用 TensorFlow/Keras 构建 MLP）**：
```python
import tensorflow as tf
model = tf.keras.Sequential([
    tf.keras.layers.Dense(64, activation='relu', input_shape=(input_dim,)),
    tf.keras.layers.Dense(32, activation='relu'),
    tf.keras.layers.Dense(num_classes, activation='softmax')
])
model.compile(optimizer='adam', loss='categorical_crossentropy', metrics=['accuracy'])
model.fit(X_train, y_train, epochs=10, batch_size=32, validation_split=0.2)
```
